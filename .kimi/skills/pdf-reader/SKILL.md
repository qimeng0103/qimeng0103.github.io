# PDF Reader Skill

> 深度阅读 PDF 文献，提取完整信息，支持文本版和扫描版

---

## 📋 功能说明

本 Skill 用于深度阅读学术论文 PDF，提取关键信息用于文献管理和报告撰写。

**核心能力：**
- 文本版 PDF：直接提取文字内容
- 扫描版 PDF：转换为图片后 OCR 识别
- 深度阅读：逐段分析，不遗漏重要信息

---

## 🔧 使用方法

### 1. 检测 PDF 类型

首先判断 PDF 是文本版还是扫描版：

```bash
# 尝试提取文本
pdftotext -f 1 -l 3 input.pdf - | head -50

# 如果输出为空或只有换行，说明是扫描版
```

### 2. 文本版 PDF 处理流程

```bash
# 提取全文（用于深度阅读）
pdftotext input.pdf full_text.txt

# 提取前5页（用于元数据识别）
pdftotext -f 1 -l 5 input.pdf metadata.txt
```

### 3. 扫描版 PDF 处理流程

```bash
# 步骤1：PDF 转为图片（前5页用于元数据，关键页面用于深度阅读）
mkdir -p /tmp/pdf_scan
pdftoppm -f 1 -l 5 -png input.pdf /tmp/pdf_scan/page

# 步骤2：使用图片阅读工具逐个识别
# 然后阅读 /tmp/pdf_scan/page-01.png 等图片

# 步骤3：清理临时文件
rm -rf /tmp/pdf_scan
```

**页面选择策略：**
- 必转页面：第1页（标题、作者）、第2-3页（摘要、引言）
- 选转页面：图表页、结论页（根据目录或缩略图判断）
- 全文处理：重要论文可转全部页面进行深度阅读

## 📖 深度阅读要求（⚠️ 严格执行）

### 禁止的行为
❌ **快速扫描** - 不能只看标题和摘要
❌ **跳过公式** - 必须阅读关键公式和推导
❌ **忽略图表** - 必须理解图注和图表内容
❌ **遗漏作者** - 必须提取完整作者列表和机构信息
❌ **跳过方法** - 必须理解实验/理论方法细节

### 必须完成的内容
✅ **完整阅读摘要** - 提取研究背景、方法、结果、意义
✅ **通读引言** - 理解研究动机和核心问题
✅ **分析方法章节** - 理解技术路线和关键步骤
✅ **识别核心结果** - 提取主要发现和创新点
✅ **记录关键数据** - 参数、数值、条件等重要信息
✅ **理解图表** - 图题、坐标轴、数据趋势

## 📝 信息提取模板

### 数学符号规范（⚠️ 强制执行）

**所有数学符号必须使用 LaTeX 格式：**

| 错误示例 | 正确 LaTeX | 说明 |
|----------|-----------|------|
| Z₀, Z_π | `$Z_0$, $Z_\pi$` | 下标用 `_` |
| π-energy | `$\pi$-energy` | 希腊字母用 `\pi` |
| 2π | `$2\pi$` | 数字与符号间空格 |
| tr(Uⁿ) | `$\operatorname{tr}(U^n)$` | 算符用 `\operatorname` |
| ⟨Γ⟩ | `$\langle\Gamma\rangle$` | 括号用 `\langle \rangle` |
| 6×6 | `$6 \times 6$` | 乘号用 `\times` |
| >50 | `$>50$` | 不等式符号用 `$` 包裹 |
| α, β | `$\alpha$, $\beta$` | 希腊字母用反斜杠 |
| d-2 | `$(d-2)$` | 数学表达式用 `$` 包裹 |
| E=mc² | `$E=mc^2$` | 上标用 `^` |

**规则：**
1. **所有数学符号**必须用 `$` 包裹（行内公式）
2. **希腊字母**用反斜杠形式（`\pi`, `\alpha`, `\beta`, `\Gamma` 等）
3. **上下标**用 `^` 和 `_`
4. **特殊函数**（tr, det, max, min 等）用 `\operatorname{}`
5. **括号**根据语义选择：`()` `[]` `\langle\rangle` `\{\}`
6. **运算符**：乘号 `\times`, 点乘 `\cdot`, 分数 `\frac{}{}`

## 🔍 扫描版 PDF 处理详细步骤

当检测到 PDF 为扫描版时，按以下流程处理：

### 步骤 1：创建临时工作目录
```bash
WORK_DIR="/tmp/pdf_scan_$(date +%s)"
mkdir -p "$WORK_DIR"
```

### 步骤 2：PDF 转图片
```bash
# 先转前5页用于基本信息提取
pdftoppm -f 1 -l 5 -png input.pdf "$WORK_DIR/page"

# 检查总页数，决定是否需要转更多页面
PDF_INFO=$(pdfinfo input.pdf 2>/dev/null || echo "Pages: 5")
TOTAL_PAGES=$(echo "$PDF_INFO" | grep "Pages:" | awk '{print $2}')

# 如果论文较短（<20页），可考虑转全文
if [ "$TOTAL_PAGES" -lt 20 ]; then
    pdftoppm -png input.pdf "$WORK_DIR/page"
fi
```

### 步骤 3：逐个识别图片
```bash
# 列出所有生成的图片
ls -v "$WORK_DIR"/page-*.png

# 按顺序阅读每张图片，提取信息
```

### 步骤 4：深度阅读策略

**逐页处理：**
1. **第1页** → 标题、作者、机构、摘要
2. **第2-3页** → 引言、背景、研究动机
3. **方法章节页** → 详细阅读实验/理论方法
4. **结果章节页** → 核心发现、数据、图表
5. **结论页** → 总结、意义、展望

**图片识别技巧：**
- 先整体浏览页面布局（栏数、图文分布）
- 按阅读顺序：标题→作者→摘要→正文
- 公式用 LaTeX 格式记录
- 图表记录坐标轴、图例、关键数据点

### 步骤 5：清理
```bash
rm -rf "$WORK_DIR"
```

## ⚠️ 重要提醒

### 阅读深度检查清单
- [ ] 是否提取了完整标题？
- [ ] 是否识别了所有作者？
- [ ] 是否理解了研究背景？
- [ ] 是否清楚了研究方法？
- [ ] 是否提取了核心结果？
- [ ] 是否理解了图表内容？
- [ ] 是否遗漏了关键公式？
- [ ] 是否记录了重要参数？

### 常见错误避免
1. **不要假设** - 不要根据标题猜测内容，必须阅读正文
2. **不要省略** - 不要省略作者列表、机构信息
3. **不要简化** - 不要简化技术方法，要记录关键细节
4. **不要跳过** - 不要跳过看起来"不重要"的段落

---

## 💡 最佳实践

1. **先读结构** - 先看目录/章节标题，了解论文组织
2. **标记重点** - 阅读时标记关键句子、数据、创新点
3. **交叉验证** - 摘要是概述，正文是细节，两者对照
4. **图表优先** - 图表往往包含最核心的结果
5. **公式理解** - 关键公式要理解其物理/数学含义
6. **记录疑问** - 不理解的术语、方法记录在 Reading Notes
